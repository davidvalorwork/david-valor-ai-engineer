QUÉ APORTO A ESTE PROYECTO:

1. Procesamiento de lenguaje natural sobre consultas bancarias
   - Integración de LLMs (OpenAI, Anthropic Claude, Llama) con prompt engineering específico para banca.
   - Clasificación de intenciones (saldos, transferencias, productos, reclamos, fraude).
   - RAG sobre documentación interna del banco para que el bot responda con información REAL y verificable, no inventada.
   - Detección de consultas sensibles que deben escalar a humano (fraude, bloqueo de tarjeta, lavado).

2. Integración con APIs bancarias
   - Diseño de capa de orquestación que aísla las APIs core del LLM (el modelo NUNCA toca la API directamente).
   - Validaciones de identidad, OTP y autenticación multifactor en flujos transaccionales.
   - Manejo de errores con fallback a operador humano cuando el bot no tiene certeza suficiente.

3. Arquitectura escalable y segura
   - Stateless backend horizontalmente escalable (Node.js / Python).
   - Cacheo de respuestas frecuentes para reducir costo de tokens en consultas repetidas (los FAQs no necesitan llamar al LLM cada vez).
   - Logging completo para auditoría, trazabilidad de decisiones del modelo y cumplimiento regulatorio.
   - Rate limiting, sanitización de input y defensa contra prompt injection.

4. Control de costos LLM
   - Estimación previa de costo por interacción y por mes según volumen esperado.
   - Routing inteligente: modelos baratos para FAQs, modelos grandes solo cuando la consulta lo justifica.

CÓMO PROPONGO TRABAJAR:

- Reunión inicial para entender alcance: qué consultas cubrir en fase 1, qué APIs bancarias existen y qué nivel de personalización requieren las respuestas.
- Documento técnico con arquitectura, estimación de costos LLM mensuales y plan de seguridad ANTES de codificar.
- Entrega por fases: MVP con consultas frecuentes resueltas → integración con APIs core → flujos transaccionales con autenticación → optimización y escalado.
- Demo funcional semanal. Repositorio documentado y handoff al equipo del banco al cierre.
